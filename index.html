📋 Scope & Applicability

This Privacy Policy applies to the Diff Viewer Chrome extension (the "Extension"), version 1.0.0 and all subsequent versions, distributed through the Chrome Web Store.

This policy does not apply to any external websites, services, or tools that you may access while using Chrome. Those are governed by their own privacy policies.

02

📊 What Data We Collect

The short answer: nothing. The Extension does not collect, transmit, store on any server, sell, share, or process any personal or non-personal data whatsoever.

Specifically, the Extension does not collect:

  • Personal identifiers (name, email, IP address, device ID)
  • The text you paste into the input panels
  • Diff results or comparison history
  • Browsing history, tab URLs, or page content
  • Usage statistics, feature interactions, or analytics
  • Crash reports or error logs sent to any server
  • Keyboard input or clipboard contents beyond the explicit copy/paste actions you initiate
  • Any information about your system, browser, or location
03

⚙️ How the Extension Works

Diff Viewer is a browser popup extension. When you click its icon, a popup window opens entirely within your browser process. All computation — text parsing, diff algorithm execution, and result rendering — occurs inside that popup window using only browser-native JavaScript APIs.

Data flow

Text you paste into the panels is held in memory (JavaScript variables and the browser's sessionStorage) for the duration of your session. It is never serialized to disk beyond sessionStorage, never sent over the network, and is automatically discarded when the popup is closed or the browser session ends.

No content scripts

The Extension does not inject any scripts into web pages you visit. It cannot read, modify, or interact with any webpage's DOM, content, or network traffic. It is invisible to every website you browse.

No background service worker with network access

The Extension does not register a background service worker. There is no persistent background process monitoring your browser activity.

Verifiable. The Extension's Content Security Policy (script-src 'self') is enforced at the browser level and technically prevents the loading of any external scripts, stylesheets, fonts, or data — even if the source code attempted to do so.

05

💾 Local Storage

The Extension uses the browser's built-in sessionStorage API to remember the text you have typed into the two input panels. This data:

  • Is stored entirely on your local device inside your browser's sandboxed storage partition for this extension
  • Is automatically deleted when the browser session ends or you close all browser windows
  • Is never synced to Chrome Sync, any cloud service, or any server
  • Cannot be read by any website or any other extension
  • Is limited in scope to the extension popup's own JavaScript context
Note on sensitive content. If you paste sensitive information (passwords, private keys, confidential documents) into Diff Viewer, that content is held in sessionStorage temporarily. While this data never leaves your device through this Extension, you are responsible for the security of your local device and browser profile.

🤝 Third Parties

Diff Viewer does not integrate with, communicate with, or share any data with any third party, including but not limited to:

  • Analytics providers (Google Analytics, Mixpanel, Amplitude, etc.)
  • Crash reporting services (Sentry, Bugsnag, Datadog, etc.)
  • Advertising networks or ad exchanges
  • Data brokers or data enrichment services
  • Cloud storage or synchronization services
  • Any API endpoints, webhooks, or remote servers of any kind

The Extension loads no external resources at runtime. All HTML, CSS, JavaScript, and image assets are bundled within the extension package itself and loaded from the extension's own origin (chrome-extension://).

👶 Children's Privacy

Diff Viewer does not collect any personal information from anyone, including children under the age of 13 (or the applicable age of digital consent in your jurisdiction). Because we collect no data at all, there is no special treatment required for children — the same zero-data guarantee applies universally.
